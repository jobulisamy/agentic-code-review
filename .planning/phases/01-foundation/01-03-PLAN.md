Create `frontend/vite.config.ts` — CRITICAL: must include Docker HMR settings:

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    host: '0.0.0.0',       // Bind to all interfaces — required inside Docker
    port: 5173,
    watch: {
      usePolling: true,     // Required on macOS + Docker volume mounts (inotify doesn't work)
    },
    hmr: {
      clientPort: 5173,     // Must match the exposed Docker host port
    },
  },
})
```

Create `frontend/Dockerfile` for dev:

```dockerfile
FROM node:20-slim

WORKDIR /app

# Copy package files first for layer caching
COPY package*.json ./
RUN npm install

# Copy remaining source (will be overridden by volume mount in docker-compose)
COPY . .

EXPOSE 5173

CMD ["npm", "run", "dev"]
```

The `COPY . .` is needed for the initial image build. In docker-compose (Plan 04), `frontend/src/` and `frontend/public/` will be mounted as volumes so live edits reflect without rebuilding the image.
